01 What it reads.

Your local Claude Code journal — already on disk.

Claude Code writes a JSONL transcript of every session to ~/.claude/projects/. Facies parses those files in place. Nothing leaves the machine. No SDK, no API key, no upload step.

  • PARSESJSONL message logs · tool calls · token accounting · model strings · timestamps · costs · cache hits
  • DERIVESStratigraphic decomposition · cache efficiency · compaction events · per-tool latency · subagent spawns
  • IGNORESCode content · prompt content · file paths in tool args · anything that could leak off-machine
  • WRITESNothing. Read-only by design.

06 Questions, asked.

FAQ

If your question isn't here, it's probably in the docs. Or we don't know the answer yet.

Does Facies send my session data anywhere?
No. The web UI is served by a local-only Vite dev server. There is no backend. There is no analytics SDK. The npm dependencies make zero outbound calls at runtime. You can verify with tcpdump or block localhost's network access entirely.
What does it cost?
Nothing. MIT licensed. github.com/ealt/facies. The only cost is the disk space your ~/.claude already occupies.
Does it work for Claude API calls outside Claude Code?
Not yet. Facies parses the JSONL transcripts that Claude Code writes to ~/.claude/projects/. If you log API calls in the same format, it'll work — but most third-party SDK use isn't logged at all. We're considering an Anthropic SDK middleware that emits compatible JSONL.
How accurate is the token decomposition?
The headline numbers (input, output, cache read, cache write) are reported verbatim from the API response and are exact. The per-layer breakdown — system vs user vs tool results — is estimated by re-tokenizing the JSONL with tiktoken. We mark estimates with ~ in every label. Variance vs. server-side count: typically <1%.
Can I share a session with a teammate?
Export → JSON or PNG of the active view. Sharing a JSONL transcript directly is not recommended — it contains your prompts, code, and tool results verbatim. Use the redaction export which strips message bodies and keeps only metrics.
